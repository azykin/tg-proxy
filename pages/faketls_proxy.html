🔒 FakeTLS Proxy Setup

Максимальная маскировка трафика под HTTPS, без рекламы

Конфигурация сервера
Домен должен указывать на IP этого сервера
Для получения Let's Encrypt сертификата
443 рекомендуется для маскировки
Для управления контейнером
Генерация секрета
FakeTLS требует секрет в формате ee + 32 hex символа + домен в hex
Ваш FakeTLS секрет
Должен начинаться с ee и содержать домен в hex

📋 Пошаговая установка

1
Установка Docker
curl -sSL https://get.docker.com/ | CHANNEL=stable bash
2
Установка Docker Compose
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose && chmod +x /usr/local/bin/docker-compose
3
Создание директории и переход в неё
mkdir -p ~/tg-proxy/faketls-proxy && cd ~/tg-proxy/faketls-proxy
4
Создание Docker Compose конфигурации
Создайте файл docker-compose.yml со следующим содержимым:
cat > docker-compose.yml << 'EOF' version: '3' services: mtproto-proxy: image: telegrammessenger/proxy:latest container_name: faketls-proxy restart: always ports: - "443:443" volumes: - ./data:/data - ./certs:/etc/letsencrypt environment: - SECRET=ваш_секрет - DOMAIN=example.com command: > sh -c "while ! [ -f /etc/letsencrypt/live/example.com/fullchain.pem ]; do echo 'Waiting for SSL certificates...'; sleep 5; done; exec mtproto-proxy -u nobody -p 8080 -H 443 -S ваш_секрет --domain example.com --tls /etc/letsencrypt/live/example.com/fullchain.pem /etc/letsencrypt/live/example.com/privkey.pem" certbot: image: certbot/certbot container_name: faketls-proxy-certbot volumes: - ./certs:/etc/letsencrypt - ./certbot-data:/var/lib/letsencrypt command: > certonly --standalone --agree-tos --non-interactive --email admin@example.com -d example.com EOF
⚠️ Заполните домен, email и секрет для формирования конфигурации
5
Запуск контейнеров
Сначала получим SSL-сертификат, затем запустим прокси:
docker-compose up -d certbot && sleep 10 && docker-compose up -d mtproto-proxy
ℹ️ Certbot автоматически получит SSL-сертификат от Let's Encrypt. Убедитесь, что порты 80 и 443 открыты и домен указывает на сервер.
6
Проверка логов
docker-compose logs -f mtproto-proxy
7
Автообновление сертификатов
Добавьте в cron обновление сертификатов (каждые 12 часов):
(crontab -l 2>/dev/null; echo "0 */12 * * * cd ~/tg-proxy/faketls-proxy && docker-compose run --rm certbot renew && docker-compose restart mtproto-proxy") | crontab -
8

🛠️ Управление прокси

⚡
Основные команды
Просмотр статуса:
docker-compose ps
Перезапуск прокси:
docker-compose restart mtproto-proxy
Просмотр логов certbot:
docker-compose logs certbot
Обновление образа:
docker-compose pull && docker-compose up -d
Остановка прокси:
docker-compose down
Полная очистка (с удалением данных и сертификатов):
docker-compose down -v && rm -rf ~/tg-proxy/faketls-proxy
Принудительное обновление сертификата:
docker-compose run --rm certbot renew --force-renew && docker-compose restart mtproto-proxy
